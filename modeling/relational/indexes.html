Внимание

Эта документация для EF Core. Для EF6.x и более ранних версий смотрите http://msdn.com/data/ef.

Примечание

В конфигурации в этом разделе применима к реляционным базам данных в целом. Методы расширения, показанные здесь станут доступны при установке провайдера реляционной базы данных (из общего пакета Microsoft.EntityFrameworkCore.Relational ).

Индексы¶

Индексы в реляционной базе данных относятся к той же концепции, что и индексы в ядре Entity Framework.

Соглашения¶

По соглашению, индексы именуются IX_<имя типа>_<имя свойства>. Для составных индексов <имя свойства> составляется из списка имен свойств, разделенных нижним подчеркиванием.

Аннотации данных¶

Индексы не могут быть конфигурированы при помощи аннотаций данных.

Fluent API¶

Вы можете использовать Fluent API для настройки имен индексов.

    class MyContext : DbContext
    {
        public DbSet<Blog> Blogs { get; set; }

        protected override void OnModelCreating(ModelBuilder modelBuilder)
        {
            modelBuilder.Entity<Blog>()
                .HasIndex(b => b.Url)
                .HasName("Index_Url");
        }
    }

    public class Blog
    {
        public int BlogId { get; set; }
        public string Url { get; set; }
    }
