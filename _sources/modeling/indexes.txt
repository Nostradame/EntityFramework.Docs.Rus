Индексы
========

Индексы являются общей концепцией во многих хранилищах данных. В то время как их реализация в хранилище данных может изменяться, они используются для наиболее эффективных операций поиска, основанных на столбце (или наборе столбцов).

.. contents:: В этой статье:
    :depth: 3

Соглашения
---------------

По соглашению, индекс создается для каждого свойства (или набора свойств), которые используются в качестве внешнего ключа.

Аннотации данных
------------------------

Индексы не могут быть созданы при помощи аннотаций данных.

Fluent API
-------------------

Вы можете использовать Fluent API чтобы указать индекс для одного свойства. По-умолчанию, индексы не уникальны.

Можно также указать, что индекс должен быть уникальным, а это означает, что никакие две сущности не могут иметь одинаковые значения для данных свойств.

Можно также указать индекс для более чем одного столбца.

.. includesamplefile:: Modeling/FluentAPI/Samples/IndexComposite.cs
        :language: c#
        :lines: 5-21
        :emphasize-lines: 7-8
        :linenos:

.. note::
    Существует только один индекс на определенный набор свойств. Если вы используете Fluent API для настройки индекса на наборе свойств, у которых уже есть определенный индекс, либо по соглашению или по предыдущей конфигурации, то вы измените определение этого индекса. Это полезно, если вы хотите дополнительно настроить индекс, который был создан по соглашению.
