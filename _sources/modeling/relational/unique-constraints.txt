Альтернативные ключи (уникальные ограничения)
=================================================

Уникальные ограничения вводятся для каждого альтернативного ключа модели.

Соглашения
---------------

По соглашению, индекс и ограничения, которые вводятся для альтернативных ключей должны быть названы ``AK_<имя типа>_<имя свойства>``. Для составных альтернативных ключей ``<имя свойства>`` составляется из списка имен свойств, разделенных нижним подчеркиванием.

Аннотации данных
------------------------

Уникальные ограничения не могут быть сконфигурированы при помощи аннотаций данных.

Fluent API
-------------------

Вы можете использовать Fluent API для настройки индексов и имен ограничений для альтернативных ключей.
