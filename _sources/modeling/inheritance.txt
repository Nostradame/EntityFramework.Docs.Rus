Наследование
==============

Наследование в модели EF используется для управления того, как наследование в классах сущностей отображается в базе данных.

.. contents:: В этой статье:
    :depth: 3

Соглашения
---------------

По соглашению, как именно наследование будет представлено в базе данных определяет провайдер базы данных . О том, как это управляется поставщиком реляционной базы данных смотрите :doc:`relational/inheritance`.

Наследование будет установлено в EF, только если два или более наследуемые типы явно включены в модель. EF не будет сканировать базу или производные типы, которые не были иным образом включены в модель. Вы можете включать типы в модели, вызвав `DbSet<TEntity>` для каждого типа в иерархии наследования.

.. includesamplefile:: Modeling/Conventions/Samples/InheritanceDbSets.cs
        :language: c#
        :lines: 5-20
        :emphasize-lines: 3-4
        :linenos:

Если вы не хотите, вызывать `DbSet<TEntity>` для одного или нескольких объектов в иерархии, вы можете использовать Fluent API, чтобы гарантировать, что они будут включены в модель.

Аннотации данных
------------------------

Вы не можете использовать Аннотации данных для настройки наследования.

Fluent API
-------------------

Fluent API для наследования зависят от поставщика базы данных, которого вы используете. О том, какую конфигурацию вы можете применить для провайдера реляционной базы данных смотрите :doc:`relational/inheritance`.
