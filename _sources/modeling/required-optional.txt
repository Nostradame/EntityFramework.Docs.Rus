Обязательные/необязательные свойства
====================================

Свойство считается необязательным, если ему может быть назначено значение ``null``. Если ``null`` не может быть назначено свойству, то оно считается обязательным.

.. contents:: В этой статье:
    :depth: 3

Соглашения
----------

По соглашениям, свойство, чей CLR тип может содержать значение null будет сконфигурировано как необязательное (``string``, ``int?``, ``byte[]``, и т.д.). Свойства, чей CLR тип не может содержать значение null будет сконфигурировано как обязательное (``int``, ``decimal``, ``bool``, и т.д.).

.. note::
    Свойство, чей CLR тип не может содержать значение null не может быть сконфигурировано как необязательное. Entity Framework всегда будет считать такие свойства обязательными.
